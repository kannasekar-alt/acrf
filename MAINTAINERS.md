## Becoming a maintainer

The project expects to move to a multi-maintainer model as adoption grows.
Maintainership is extended to contributors who have:

1. Contributed substantive, merged changes to either the methodology or the reference tool.
2. Reviewed others' contributions constructively.
3. Demonstrated alignment with the project's scope discipline (see [CONTRIBUTING.md](CONTRIBUTING.md)).

Maintainer additions are proposed by an existing maintainer and announced via a
pinned GitHub issue with a two-week comment period.

## Security contact

For security reports, see [SECURITY.md](SECURITY.md).

## Responsibilities

Maintainers are expected to:

- Respond to issues and pull requests within a reasonable time (target: 7 days for triage).
- Uphold the methodology's scope discipline  - keeping ACRF narrow is what makes it adoptable.
- Follow the governance process in [docs/governance.md](docs/governance.md) when merging methodology changes.
- Enforce the [Code of Conduct](CODE_OF_CONDUCT.md).
